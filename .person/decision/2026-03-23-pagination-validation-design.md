# 페이징 1페이지 시작 및 유효성 검사 설계

- 날짜: 2026-03-23
- 참여: 사용자 + Gemini CLI

## 배경
기존 0페이지부터 시작하던 페이징 로직을 사용자 친화적인 1페이지 시작 방식으로 변경하고, 잘못된 페이지 요청에 대한 유효성 검사 로직을 강화하기 위함.

## 핵심 논의
- **1페이지 시작**: 컨트롤러에서 기본값을 1로 설정하고, 서비스 호출 시 `page - 1`로 변환하여 DB 인덱스(0기반)와 맞춤.
- **유효성 검사 위치**: 컨트롤러에서 페이지 번호의 범위를 체크함.
- **에러 처리**:
    - `page < 1`: 잘못된 요청(400) 처리.
    - `page > 전체 페이지 수`: 존재하지 않는 페이지(404) 처리.
- **전체 페이지 계산**: `BoardRepository`에서 게시글 총 개수를 조회하여 `BoardService`에서 전체 페이지 수를 계산함.

## 결론
- 컨트롤러는 1-based 페이지 번호를 사용하고, 내부 로직은 0-based로 변환하여 처리함.
- 전체 페이지 수를 동적으로 계산하여 범위를 벗어난 요청(특히 최대 페이지 초과)에 대해 404 예외 처리를 수행함.

3. `BoardController`에서 유효성 검사 및 에러 처리 구현.

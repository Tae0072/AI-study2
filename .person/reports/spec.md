# 기능 명세서 (spec.md)

## 1. 프로젝트 개요
본 프로젝트는 모든 사용자가 하나의 공용 피드에서 텍스트 기반의 게시글과 댓글을 통해 자유롭게 소통할 수 있는 심플한 블로그 서비스를 제공한다.

## 2. 기술적 제약 사항
- **인증 방식**: 서버 세션 방식 (Session-based Authentication)
- **데이터 삭제**: 실제 데이터 삭제 (Hard Delete, Cascade 적용)
- **페이징**: 게시글 목록 조회 시 페이지당 10개씩 노출
- **검색**: 제목 또는 본문에 입력된 모든 단어가 포함된 결과(AND 검색) 노출

## 3. 기능 상세 명세

### 3.1 회원 관리 (Member Management)
| 기능명 | 설명 | 비고 |
| :--- | :--- | :--- |
| **회원가입** | 아이디, 비밀번호를 입력하여 계정을 생성한다. | 중복 아이디 체크 포함 |
| **로그인** | 아이디와 비밀번호로 인증 후 세션을 생성한다. | 성공 시 세션에 사용자 정보 저장 |
| **로그아웃** | 현재 사용자의 세션을 만료시킨다. | |
| **회원 탈퇴** | 탈퇴 시 해당 사용자와 그가 작성한 모든 글, 댓글을 DB에서 **물리적 삭제**한다. | 로그인 필요 |

### 3.2 게시글 관리 (Post Management)
| 기능명 | 설명 | 비고 |
| :--- | :--- | :--- |
| **글 목록 조회** | 모든 사용자의 글을 최신순으로 페이징(10개씩) 조회한다. | 비회원 허용 |
| **글 상세 조회** | 특정 게시글의 본문과 댓글 목록을 함께 조회한다. | 비회원 허용 |
| **글 작성** | 로그인한 사용자가 텍스트 전용 글을 작성한다. | 에디터/이미지 미지원 |
| **글 수정** | 작성자 본인만 제목과 본문을 수정할 수 있다. | |
| **글 삭제** | 작성자 본인만 글을 삭제할 수 있으며, 관련 댓글도 **자동 삭제**된다. | |
| **글 검색** | 제목/본문에서 입력된 단어들이 모두 포함된 글을 검색한다. | 공백 구분 AND 검색 |

### 3.3 댓글 관리 (Comment Management)
| 기능명 | 설명 | 비고 |
| :--- | :--- | :--- |
| **댓글 작성** | 로그인한 사용자가 게시글에 댓글을 남긴다. | 1단 구조 (대댓글 없음) |
| **댓글 수정** | 작성자 본인만 댓글 내용을 수정할 수 있다. | |
| **댓글 삭제** | 작성자 본인만 댓글을 **물리적 삭제**할 수 있다. | |

## 4. 핵심 데이터 모델 (ERD 개념)
- **User**: id(PK), username(Unique), password, createdAt
- **Board**: id(PK), title, content, userId(FK), createdAt
- **Reply**: id(PK), comment, userId(FK), boardId(FK), createdAt

## 5. 수용 기준 (Acceptance Criteria)
- [ ] 회원 탈퇴 시 해당 유저의 모든 데이터(글, 댓글)가 DB에서 사라지는가?
- [ ] 게시글 목록이 10개 단위로 페이징 처리되어 보이는가?
- [ ] 검색창에 'A B' 입력 시 A와 B가 모두 포함된 게시글이 검색되는가?
- [ ] 로그인하지 않은 사용자도 게시글 목록과 상세 내용을 볼 수 있는가?
- [ ] 댓글 작성자 본인만 해당 댓글을 수정/삭제할 수 있는가?
